## 1. 测试策略

### 1.1 测试目标

| 目标维度 | 具体目标 | 衡量指标 |
|---------|---------|---------|
| **功能正确性** | 验证55个功能函数按规格正确实现 | 测试用例通过率 ≥ 95% |
| **数据一致性** | 确保多数据源同步（eClass/WebSAMS/生物识别） | 数据同步准确率 ≥ 99.5% |
| **安全合规** | 验证PDPO合规、双人见证、审计追踪 | 安全测试用例100%通过 |
| **性能基准** | 验证系统响应时间满足架构目标 | P95 < 500ms (CRUD) |
| **多语言支持** | 验证zh-HK/zh-CN/en三语言完整覆盖 | 界面文本覆盖率100% |

### 1.2 测试层级

```
┌─────────────────────────────────────────────────────────────────┐
│  Level 5: 验收测试 (UAT)                                        │
│  - 校务处真实用户场景验证                                        │
│  - 业务流程端到端验证                                            │
├─────────────────────────────────────────────────────────────────┤
│  Level 4: 系统测试 (System Test)                                │
│  - 全模块集成测试                                               │
│  - 性能/安全/兼容性测试                                          │
├─────────────────────────────────────────────────────────────────┤
│  Level 3: 集成测试 (Integration Test)                           │
│  - 服务间接口测试 (API Contract)                                │
│  - 数据库事务一致性测试                                          │
├─────────────────────────────────────────────────────────────────┤
│  Level 2: 模块测试 (Module Test) ★ 本方案重点                    │
│  - 9大模块独立功能验证                                           │
│  - 55个功能函数全覆盖                                            │
├─────────────────────────────────────────────────────────────────┤
│  Level 1: 单元测试 (Unit Test)                                  │
│  - 代码级函数测试                                               │
│  - 自动化覆盖率 ≥ 80%                                            │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 测试类型分配

| 测试类型 | 占比 | 执行方式 | 负责角色 |
|---------|------|---------|---------|
| 功能测试 | 50% | 手工+自动化 | QA工程师 |
| 接口测试 | 15% | 自动化 | 开发工程师 |
| 性能测试 | 10% | 自动化 | 性能测试工程师 |
| 安全测试 | 10% | 手工+自动化 | 安全测试工程师 |
| 兼容性测试 | 10% | 手工 | QA工程师 |
| 回归测试 | 5% | 自动化 | CI/CD流水线 |

### 1.4 测试用例设计方法

| 功能类别 | 设计方法 | 用例配比 |
|---------|---------|---------|
| **业务流程类** (如请假审批、收费) | 场景法 + 状态迁移 | 正常:异常:边界 = 1:1:1 |
| **数据管理类** (如学生信息、成绩) | 等价类 + 边界值 | 正常:异常:边界 = 1:2:1 |
| **权限控制类** (如RBAC/ABAC) | 判定表 + 正交试验 | 角色×操作×数据范围组合 |
| **接口集成类** (如WebSAMS同步) | 契约测试 + 异常注入 | 成功场景:失败处理:超时 = 1:1:1 |
| **AI功能类** (如OCR/NLU) | 统计采样 + 对抗测试 | 准确率评估 + 边界case |
