# DoD（Definition of Done）完成标准

本文档定义了功能、迭代和版本级别的完成标准，确保交付质量。

---

## 功能级 DoD

每个功能开发完成前必须满足以下标准：

- [ ] **功能实现完整**（按SPEC要求）
- [ ] **代码审查通过**（CHECKER审查）
- [ ] **单元测试覆盖率≥70%**
- [ ] **集成测试通过**
- [ ] **P0/P1缺陷已修复**
- [ ] **文档已更新**（API文档、用户手册）
- [ ] **PM验收通过**

---

## 迭代级 DoD

每个迭代（Sprint）结束时必须满足以下标准：

- [ ] **计划功能全部完成**
- [ ] **测试覆盖率达标**
- [ ] **所有缺陷清零**
- [ ] **性能指标达标**
- [ ] **安全扫描无高危问题**
- [ ] **UAT通过**

---

## 版本级 DoD

每个版本发布前必须满足以下标准：

- [ ] **所有功能完成并验收**
- [ ] **系统稳定运行**（无重大Bug）
- [ ] **文档齐全**（用户手册、运维手册）
- [ ] **部署脚本验证通过**
- [ ] **回滚方案准备完毕**

---

## 使用说明

1. **功能开发**：开发人员在完成每个功能后，对照"功能级DoD"自查，确保所有项勾选后再提交测试。
2. **迭代结束**：Sprint结束时，团队对照"迭代级DoD"评审，确保所有交付物符合标准。
3. **版本发布**：Release前，PM对照"版本级DoD"进行最终验收。

---

## 审批流程

| 级别 | 审批人 | 说明 |
|------|--------|------|
| 功能级 | Tech Lead + PM | 代码审查通过后，PM验收功能 |
| 迭代级 | PM + QA | 迭代结束评审会确认 |
| 版本级 | PM + DEVOPS | 发布前最终确认 |
