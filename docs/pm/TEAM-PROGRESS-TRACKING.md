## 🎯 设计原则

1. **快速迭代** - AI团队工作节奏快，12h无更新即预警
2. **独立报告** - 每个角色独立提交进度，不被PM掩盖
3. **透明可见** - 所有角色进度公开，相互监督

## 📊 停滞报警机制

### 报警级别

| 级别 | 时间阈值 | 行动 | 通知对象 |
|------|----------|------|----------|
| 🟡 警告 | 12h无报告 | PM询问状态 | PM + 角色本人 |
| 🔴 严重 | 24h无报告 | 紧急介入 | PM + 用户 |
| ⚫ 危机 | 36h无报告 | 角色替换 | PM + 用户 |

### 自动检测脚本

```bash
#!/bin/bash
# check-role-activity.sh - 检查各角色活动状态

ROLE_REPORTS="/workspace/projects/workspace/docs/pm/role-reports"

for role in DEV1 DEV2 DEV3 DEV-FRONTEND QA1 QA2 CHECKER DEVOPS; do
    latest_report=$(ls -t ${ROLE_REPORTS}/${role}*.md 2>/dev/null | head -1)

    if [ -z "$latest_report" ]; then
        echo "🔴 $role - 无报告记录"
    else
        report_time=$(stat -c %Y "$latest_report")
        current_time=$(date +%s)
        hours_ago=$(( (current_time - report_time) / 3600 ))

        if [ $hours_ago -gt 36 ]; then
            echo "⚫ $role - $hours_ago小时无报告 (危机)"
        elif [ $hours_ago -gt 24 ]; then
            echo "🔴 $role - $hours_ago小时无报告 (严重)"
        elif [ $hours_ago -gt 12 ]; then
            echo "🟡 $role - $hours_ago小时无报告 (警告)"
        else
            echo "🟢 $role - $hours_ago小时前报告 (正常)"
        fi
    fi
done
```

### 报警通知流程

```
角色12h无报告
    ↓
脚本检测 → 发送提醒至角色
    ↓
角色24h无报告
    ↓
脚本检测 → 发送警报至PM + WeChat通知用户
    ↓
角色36h无报告
    ↓
脚本检测 → 触发紧急会议 + 考虑角色替换
```
