# PM 工作模式 - 7x24持续工作

**更新**: 2026-06-14
**原则**: AI团队7x24，不需定时，按优先级和依赖关系持续工作

---

## 🎯 核心原则

### 1. 持续工作，不需定时

- ❌ 不等"10:00/14:00/18:00"
- ❌ 不等用户提醒
- ❌ 不等外部触发
- ✅ 发现问题立即解决
- ✅ 持续工作直到完成

### 2. 按优先级处理

**优先级定义**:
- **P0**: 阻塞性问题（CI失败、核心功能不可用）
- **P1**: 高价值功能（客户急需）
- **P2**: 一般功能（计划内）
- **P3**: 优化/改进

**规则**: 始终处理最高优先级任务

### 3. 识别依赖关系

**依赖类型**:
- **阻塞依赖**: A完成后B才能开始
- **并行依赖**: A和B可以同时进行
- **串行依赖**: A→B→C顺序执行

**规则**: 
- 无依赖的任务并行执行
- 有依赖的任务按顺序执行

### 4. 各角色按规则执行

**DEV角色**:
- 接收任务 → 开发 → 提交 → 报告
- 不等待，随时可工作

**QA角色**:
- 代码就绪 → 执行测试 → 报告Bug
- 有代码立即测试

**DEVOPS角色**:
- CI失败 → 分析 → 修复
- 持续监控

**CHECKER角色**:
- 代码提交 → 审查 → 报告
- 随时待命

## 🔄 持续工作流程

```
发现任务/问题
    ↓
评估优先级
    ↓
检查依赖
    ↓
分配角色（或自己处理）
    ↓
执行（并行或顺序）
    ↓
完成 → 下一任务
    ↓（如有阻塞）
协调解决 → 继续执行
```
