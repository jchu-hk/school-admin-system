# PM主动监控与任务分配机制

**创建**: 2026-06-14 21:49
**目的**: PM主动监控各角色工作，预先分配任务

---

## 1. 问题分析

### 当前问题

| 问题 | 原因 |
|------|------|
| Agents完成未commit | 未明确要求报告机制 |
| 任务积压 | PM未主动分配 |
| 状态不同步 | 无定期检查 |
| 等待时间长 | 有任务但未分配 |

### 根本原因

1. **缺乏主动监控**: PM等待报告，不主动检查
2. **缺乏预警机制**: 任务超时未被发现
3. **缺乏交接流程**: 完成→验收流程不清晰

### 2.3 任务就绪检查

**任务可以开始的条件**:

```yaml
P0/P1任务:
  ✓ CHECKER已批准需求 (approved标签)
  ✓ ARCH已批准设计 (如需要)
  ✓ PM已分配负责人
  ✓ 依赖任务已完成

P2/P3任务:
  ✓ 需求描述清晰
  ✓ PM已分配负责人
  ✓ 依赖任务已完成
```

**检查流程**:
```
获取所有Open Issues
    ↓
筛选未分配的任务
    ↓
检查依赖是否满足
    ↓
检查需求是否清晰
    ↓
分配给合适Agent
    ↓
设置deadline
```

## 3. 主动分配规则

### 3.1 任务分配触发

| 触发条件 | 动作 |
|----------|------|
| 新Issue创建 | PM评估优先级和分配 |
| 依赖完成 | 立即检查是否可以开始 |
| Agent空闲 | 检查待分配任务 |
| 定时检查 | 每30分钟全面检查 |

### 3.2 Agent分配规则

```yaml
任务类型 → Agent:
  后端开发 → DEV
  前端开发 → UI
  需求分析 → REQ
  架构设计 → ARCH
  测试验收 → QA
  代码审查 → CHECKER
  备份/监控 → OPS
  CI/CD → DEVOPS
```

### 3.3 分配优先级

```
1. P0任务 (无依赖) → 立即分配
2. P0任务 (有依赖) → 等依赖完成后立即分配
3. P1任务 (无依赖) → 立即分配
4. P1任务 (有依赖) → 依赖完成后分配
5. P2任务 → 资源允许时分配
6. P3任务 → 按迭代计划分配
```

## 4. 超时预警

### 4.1 时间阈值

| 任务类型 | 预警时间 | 最大时间 |
|----------|----------|----------|
| P0 | 30分钟 | 2小时 |
| P1 | 1小时 | 4小时 |
| P2 | 2小时 | 8小时 |
| P3 | 4小时 | 24小时 |
| QA验收 | 1小时 | 4小时 |
| CHECKER审查 | 1小时 | 4小时 |

### 4.2 超时处理

```yaml
超过预警时间:
  → Agent未完成 → 检查进度 + 要求更新

超过最大时间:
  → 未完成 → 创建阻塞Issue
  → PM介入协调
  → 可能需要更换Agent

超过2倍最大时间:
  → 升级处理
  → 评估是否拆分任务
```
