## 工作流程 (QA + CHECKER)

### 开发完成流程
```
DEV完成 → 添加ready-for-review标签 → GitHub自动通知QA
QA验收 → 添加passed/failed标签 → GitHub自动通知CHECKER
CHECKER审批 → 添加approved/changes-requested标签 → 通知PM合并
PM合并 → 关闭Issue
```

### 新任务分派流程
```
REQ需求分析 → ARCH架构审查 → CHECKER审批 → PM分配DEV开发
```

## PM主动监控机制

### 5分钟检查清单
1. 检查运行中的Agents
2. 检查ready-for-review → 分配QA
3. 检查passed → 分配CHECKER

## 教训

### PM工作原则
1. **持续工作**: 不等外部触发，发现问题立即解决
2. **主动监控**: 每5分钟检查Agents和GitHub状态
3. **及时同步**: 完成工作立即commit和更新Issue
4. **报告透明**: 所有状态变化通过GitHub记录

### 7x24工作模式
- AI团队7x24持续工作
- 按优先级处理任务
- 识别依赖关系
- 各角色按规则执行
